# H5 商城 Implementation Plan

> **For Hermes:** 使用 subagent-driven-development 逐任务实现。

**Goal:** 为流通处构建面向消费者的 H5 移动商城（浏览商品、加购、下单、查订单）

**Architecture:** 独立 React 应用 (`h5-mall/`) ，直连 SDB 读数据 + 轻量 API 写订单，移动端优先，零第三方 UI 库

**Tech Stack:** React 19 + TypeScript + Vite，与 admin-react 相同技术栈

**Deploy:** `/var/www/v5-h5/`，路径 `/h5/`

---

## 架构决策

### 数据读写策略

| 操作 | 方式 | 说明 |
|------|------|------|
| 商品列表/详情 | 前端直连 SDB（只读 scope） | 与 admin 同模式，无需 API 中转 |
| 轮播图/推荐/公告 | 前端直连 SDB | 只读 |
| 下单 | `POST /api/order` → 服务端写 SDB | 防止客户端直接写库 |
| 查订单 | `GET /api/orders?phone=xxx` | 简单查询 |

### 为什么不用全量 API 代理

- admin 已验证前端直连 SDB 的可行性（3个月生产运行）
- H5 只读操作无需额外保护
- 下单是唯一需要服务端参与的操作（生成订单号、防重复）

### 顾客身份

- 浏览：无需登录
- 下单：填写姓名 + 手机号，自动创建/匹配 `customer` 记录
- 查订单：输入手机号查询

### 购物车

- 存 `localStorage`，无需服务端
- 结构：`{ variantId, quantity, unitPrice }[]`

---

## 页面结构（7 页）

```
/                 → 首页（轮播→分类入口→推荐商品→公告）
/products         → 商品列表（分类筛选 + 搜索）
/product/:id      → 商品详情（图、变体、价格、加购）
/cart             → 购物车（编辑数量、去结算）
/checkout         → 下单（填信息、确认、提交）
/order/:id        → 下单成功页
/orders           → 订单查询（手机号查历史订单）
/activity/:id     → 活动详情（活动报名专用）
```

## 数据库准备（SDB）

### 新增：H5 只读 Scope

```sql
DEFINE SCOPE h5_public SESSION 24h
  SIGNUP (CREATE user SET name = 'h5_visitor', role = '店员')
  SIGNIN (SELECT * FROM user WHERE name = 'h5_visitor');
```

然后给各表加只读 PERMISSIONS（product, product_category, product_variant, pricing, store_inventory, carousel, featured_product, store_info, announcement）。

### 新增：服务端下单 API（Python）

在 `server/` 目录下新建 `order_api.py`，暴露两个端点：

| 端点 | 说明 |
|------|------|
| `POST /api/order` | 创建订单。Body: `{customer: {name, phone}, items: [{variantId, quantity}], payment_method, notes}` |
| `GET /api/orders?phone=xxx` | 按手机号查订单 |

## 逐任务实现（共 24 个 Task）

### Phase 1：项目骨架（Task 1-4）

#### Task 1：创建 h5-mall 项目

```bash
cd rufazao-distribution
npm create vite@latest h5-mall -- --template react-ts
cd h5-mall
npm install react-router-dom
```

#### Task 2：配置 Vite（base path + alias）

`vite.config.ts`：base: `/h5/`，alias `@` → `src`，注入 `__COMMIT_HASH__`

#### Task 3：SDB 连接层

复制 admin-react 的 `src/lib/sdb.ts`，改造为：
- 使用 H5 只读 scope 登录（匿名）
- 暴露 `sdbQuery(sql)` 函数

#### Task 4：路由骨架 + App 壳

`App.tsx`：BrowserRouter + 7 条路由 + 移动端布局壳（顶部导航栏 + 底部 TabBar + 中间内容区）

### Phase 2：首页（Task 5-9）

水平滑动轮播，自动播放 3s，圆点指示器

#### Task 8：推荐商品组件

2 列网格，商品卡片（图片、名称、价格）

#### Task 9：公告栏组件

滚动公告条

### Phase 3：商品列表 + 详情（Task 10-14）

#### Task 10：商品列表页

- 按分类筛选：`SELECT * FROM product WHERE is_listed=true AND category=<id> FETCH category`
- 搜索：`SELECT * FROM product WHERE is_listed=true AND name CONTAINS $q`
- 向量搜索（可选）：`SELECT * FROM product WHERE content_embedding <|2|> $embedding`
- 2 列网格展示

#### Task 11：商品详情页

- 主图 + 详情图（图片画廊）
- 商品名、描述、分类
- 变体列表（从 product_variant 关联 pricing 取售价）
- 库存显示
- 「加入购物车」按钮（选变体 + 数量）

#### Task 12：图片画廊组件

- 主图大图，缩略图列表可切换
- 点击放大全屏查看

#### Task 13：变体选择器

- 列出该 SPU 的所有 variant
- 显示 SKU 名 + 售价 + 库存
- 选中后高亮

#### Task 14：加入购物车交互

- 选变体 → 选数量（+/- 按钮，最少 1，最多库存数）
- 点击「加入购物车」
- Toast 提示 + 购物车角标 +1

---

### Phase 4：购物车 + 下单（Task 15-20）

#### Task 15：购物车页

- 从 localStorage 读购物车数据
- 列表展示：商品名、变体、单价、数量（可编辑）、小计
- 全选/单选、合计金额
- 「去结算」按钮

#### Task 16：购物车状态管理

`src/stores/cart.ts` — useCart hook：
- `addItem(variantId, quantity, unitPrice, productName, variantName)`
- `removeItem(variantId)`
- `updateQuantity(variantId, quantity)`
- `clearCart()`
- `totalAmount` 计算属性
- 数据持久化到 localStorage

#### Task 17：下单页（Checkout）

- 顾客信息表单：姓名、手机号（必填）
- 支付方式：在线支付 / 线下付款（Radio）
- 备注（选填）
- 订单摘要（商品列表 + 金额）
- 「确认下单」按钮

#### Task 18：下单 API（服务端）

`server/order_api.py`：
- 接收 POST `/api/order` 
- 生成订单号 `SO` + 日期 + 序号
- upsert customer（按手机号匹配或新建）
- INSERT sales_order + order_items
- 扣减库存（UPDATE store_inventory SET quantity -= n）
- 返回订单详情

#### Task 19：下单页提交逻辑

- 表单校验（姓名、手机号非空）
- 调用 `POST /api/order`
- 成功后清空购物车、跳转 `/order/:id`

#### Task 20：下单成功页

- 显示订单号、金额、支付方式
- 线下付款提示文案
- 「继续逛逛」「查看订单」按钮

---

### Phase 5：订单查询 + 活动（Task 21-24）

#### Task 21：订单查询页

- 输入手机号 → 查询订单列表
- `GET /api/orders?phone=xxx`
- 订单卡片：订单号、时间、金额、状态（带颜色标签）
- 点击查看订单详情

#### Task 22：订单查询 API

`GET /api/orders?phone=xxx`：
- 按 phone 查 customer → 按 customer 查 sales_order
- FETCH order_item + variant + product
- 返回订单列表（含商品明细）

#### Task 23：活动详情页

- 活动商品（product_type='活动'）专用详情
- 显示：开始/截止日期 或 周期说明
- 报名上限 + 已报名数
- 「立即报名」按钮 → 走下单流程（0 元订单或付费订单）

#### Task 24：底部导航栏

固定底部的 TabBar（4 个 Tab）：
- 首页 🏠
- 分类 📂
- 购物车 🛒（角标显示数量）
- 订单 📋

扩展现有 `deploy.sh`，增加 H5 部署逻辑：

```bash
# H5 Mall 部署
scp -r h5-mall/dist/* ubuntu@VPS:/var/www/v5-h5/
```

```
handle_path /h5/* {
    root * /var/www/v5-h5
    file_server
}
```

### 服务端 API 部署

`order_api.py` 通过 systemd 或 Caddy reverse_proxy 运行。

## 设计规范

### 移动端优先

- 最大宽度 480px 居中
- 字体大小 14-16px（适合手机阅读）
- 触控区域 ≥ 44px
- 底部留出 TabBar 安全区

### 配色

- 主色：`#c41e3a`（佛系红/朱砂色）
- 背景：`#f5f0eb`（米白/宣纸色）
- 卡片：`#fff`
- 文字：`#333`（正文）`#999`（次要）

### 组件原则

- 零第三方 UI 库（与 admin 一致）
- 纯 CSS，善用 flexbox
- 图片懒加载
- 过渡动画（opacity + transform）

---

## 风险与对策

| 风险 | 对策 |
|------|------|
| SDB 只读 scope 配置复杂 | 降级方案：H5 前端通过 `/api/` 代理所有 SDB 查询 |
| 移动端兼容性 | 只用标准 CSS（flexbox, grid），不依赖实验性特性 |
| 图片加载慢 | 图片懒加载 + Skeleton 占位 |
| 库存并发（超卖） | 下单时 SDB 做乐观锁检查 `quantity >= n` |

## 不变更项

- ❌ 不新增 DB 表（已有 13 张表足够）
- ❌ 不修改 admin 前端
- ❌ 不做微信小程序（仅 H5 网页）
- ❌ 不接入真实支付（仅标记支付方式）
- ❌ 不用第三方 UI 库
